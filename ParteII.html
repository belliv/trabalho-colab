Pipeline



Nesta etapa será criado um Pipeline utilizando a biblioteca Scikit-Learn. O Pipeline reúne, em um único fluxo, a preparação dos dados (seleção de atributos) e o treinamento do algoritmo Random Forest Regressor.

Essa abordagem facilita a organização do código e evita que etapas importantes sejam esquecidas durante o treinamento do modelo.
In [4]:
# Criação do Pipeline

pipeline = Pipeline([
    ("selecao", SelectKBest(score_func=f_regression, k=5)),
    ("modelo", RandomForestRegressor(
        n_estimators=100,
        random_state=42
    ))
])

# Treinando o Pipeline

pipeline.fit(X_train, y_train)

print("Pipeline treinado com sucesso!")
Pipeline treinado com sucesso!

Avaliação do modelo



Para avaliar o desempenho do modelo serão utilizadas duas métricas de regressão:

- MAE (Mean Absolute Error)
- RMSE (Root Mean Squared Error)

Essas métricas permitem verificar o erro médio das previsões realizadas pelo modelo.
In [6]:
from sklearn.metrics import mean_absolute_error, mean_squared_error
import numpy as np

mae = mean_absolute_error(y_test, predicoes)
rmse = np.sqrt(mean_squared_error(y_test, predicoes))

print(f"MAE: {mae:.3f}")
print(f"RMSE: {rmse:.3f}")
MAE: 1.311 RMSE: 1.789
 

Conclusão



Nesta segunda parte do trabalho foi utilizado um Pipeline para automatizar as etapas de seleção de atributos e treinamento do modelo Random Forest Regressor.

As métricas MAE e RMSE permitiram avaliar o desempenho do modelo utilizando a base de testes. Quanto menores forem esses valores, menor será o erro das previsões realizadas.

Os resultados obtidos indicam que o modelo apresenta um desempenho satisfatório para este conjunto de dados, sendo capaz de estimar a pressão sonora com um nível de erro considerado aceitável. Entretanto, esse desempenho poderia ser melhorado por meio do ajuste dos hiperparâmetros do algoritmo, da utilização de outras técnicas de seleção de atributos ou da comparação com diferentes algoritmos de regressão.
